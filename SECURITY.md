# Security Policy

## Supported Versions

This is an actively maintained starter. We aim to promptly address security issues on the `main` branch.

## Reporting a Vulnerability

If you believe you’ve found a security vulnerability, please:

1. Do not open a public issue.
2. Email or DM the maintainer privately, or open a private security advisory on GitHub if available.
3. Provide a minimal reproduction, affected versions, and impact assessment.

We will acknowledge your report, investigate, and provide a timeline for a fix. Thank you for helping keep the project and its users safe.
